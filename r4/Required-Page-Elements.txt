R4 User Interface Standards

Required Page Elements
Revision Date: September 5, 2025
Software project standards encompass layout, meta tags, and design guidelines that
improve usability for our customers. For example, these standards include the use of alt
attributes for images that convey meaning, labeling of file download sizes, and the
inclusion of unique and accurate titles and descriptions.

Authenticated R4 Systems (https://[systemacronym].ctc.ca.gov)
All pages in an authenticated R4 system should have a header, left navigation, footer,
breadcrumbs, and a browser page title that follow the standards outlined below, unless
otherwise discussed with the Systems Architect and Software Development Manager.
An authenticated R4 system can include several different portals. The standards listed
below apply to all authenticated R4 portals, unless otherwise indicated.

Header


(UI-100) The Commission on Teacher Credentialing logo should be to the far left in

the header above the left navigation.


(UI-101) The system acronym should be displayed left-aligned in the header (this is

the R4 “Acronym” field for a Portal). See the Application Profile Portal Naming
Standards for more information.


(UI-102) A link to Reports, Help, and the user profile (in this order) should be

right-aligned.

Left Navigation


(UI-103) The first link should be titled Home and should be linked to the system’s

home page.


(UI-104) All other links should be after the Home link in order of most use.

Footer


(UI-105) The first line in the footer should be the full name of the system (with no

system acronym).


(UI-106) Immediately following the system’s full name on the first line is the URL

of the system (https://[systemacronym].ctc.ca.gov/). The URL should not be
hyperlinked.


(UI-107) The second line in the footer should have the word “Support:” followed by

CRISP (hyperlinked to https://crisp.ctc.ca.gov/) (only for Staff Portals). For
audience-specific authenticated Portals, discuss with the Systems Architect and
Software Development Manager.


(UI-108) The third and last line in the footer is “Authorized Use Only” followed by

“Privacy Policy”, which is linked to the Commission’s Privacy Policy
(https://www.ctc.ca.gov/site-information/privacy-policy).


(UI-109) The build number is listed in the footer to the far right. Use the word

“Build:” followed by the build/version number of the system.

Breadcrumbs

page.


(UI-111) Breadcrumbs should be displayed on the same line as the Page Title, but

right-aligned.


which is never hyperlinked.

Meetings”.

Non-Authenticated R4 Systems (Open to Public)
(http://xxxx.ctc.ca.gov/)
All non-authenticated R4 systems and pages should have a header, left navigation, footer,
and a browser page title that follow these standards:

Header


(UI-121) The Commission on Teacher Credentialing logo should to the far left in the

header.


(UI-122) A link to CTC Home should be right-aligned.

Navigation


(UI-123) The navigation will be immediately below the header, with the menu

items being horizontal.

Footer


(UI-124) The first line in the footer should be the full name of the system (with no

system acronym).


(UI-125) Immediately following the system’s full name on the first line is the URL

of the system (https://xxxx.ctc.ca.gov/). The URL should not be hyperlinked.


(UI-126) The second line in the footer should have the word “Contact: ” followed by

the name of a business program or the email address of a shared mailbox that is
related to the Portal. The business program name or the email address should be
displayed without being hyperlinked, followed by a comma. After the comma, the
email address of the business program is displayed and hyperlinked. Contact the
Systems Architect and Software Development Manager if there is any uncertainty.


(UI-127) The third and last line in the footer is “Authorized Use Only” followed by

“Privacy Policy”, which is linked to the Commission’s Privacy Policy
(https://www.ctc.ca.gov/site-information/privacy-policy).

(UI-128) The build number is listed in the footer to the far right. Use the word

Meta Tags
Meta tags provide metadata about the webpage. This information is not displayed on the
page but is used by search engines or other web services.
The following meta tags must be included in every web page that is published to the
public Commission website (https://xxxx.ctc.ca.gov):


(UI-129) Description (<meta name=”keywords” content=””)–description of the

content being displayed on the page.


(UI-130) DC.date.created (<meta name=”DC.date.created” scheme=”ISO08601”

content=””>)–the date the page was created. The date should be in the following
format: year-month-day (i.e., 2021-05-06).


(UI-131) DC.date.modified (<meta name=”DC.date.modified” scheme=”ISO08601”

content=””>)–the date the page was modified. The date should be in the following
format: year-month-day (i.e., 2021-05-06).
